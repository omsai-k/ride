# Workflow Breakdown for RiderComm Architecture

This folder contains prompt-based AI workflows for developing each component of the RiderComm architecture. Each file corresponds to a distinct subsystem and describes:

- Component purpose
- Input assumptions
- Development steps
- Expected deliverables
- Validation criteria
- Handoffs to the next component

Use these files as task packages for AI agents or chat sessions to implement the project in parallel or sequentially.
